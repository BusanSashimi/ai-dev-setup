# 모듈별 상수 관리 가이드라인

## 📚 개요

이 문서는 프론트엔드 프로젝트에서 상수(Constants)를 모듈별로 체계적으로 관리하기 위한 가이드라인을 제공합니다. 일관된 상수 관리를 통해 코드의 가독성, 유지보수성을 향상시키고 중복 코드를 방지할 수 있습니다.

## 🎯 기본 원칙

### 1. 모듈별 분리

- 각 모듈은 독립적인 `constants` 폴더를 가집니다
- 기능별로 상수 파일을 분리하여 관리합니다
- 파일명은 `[기능명].constants.ts` 형식을 따릅니다

### 2. 일관된 인터페이스

- 모든 상수 클래스는 동일한 베이스 클래스를 상속받습니다
- 표준화된 메서드를 제공합니다 (`getName`, `getOptions` 등)
- TypeScript의 타입 안전성을 보장합니다

### 3. 네이밍 컨벤션

- 상수 클래스명: PascalCase (예: `TaxiClientState`)
- 상수 값: SCREAMING_SNAKE_CASE (예: `OPEN`, `CLOSE`)
- 파일명: kebab-case (예: `client-state.constants.ts`)

## 🏗️ 베이스 클래스 구현

### BaseCodeConstants 클래스

모든 상수 클래스가 상속받는 베이스 클래스입니다.

```typescript
// src/modules/common/constants/base-code.constants.ts
import { Utils } from '@/modules/common/utils/utils';

export interface SelectOption {
  text: string;
  value: string;
}

export abstract class BaseCodeConstants {
  abstract readonly names: Record<string, string>;

  getName(code: string | number): string {
    return this.names[String(code)] || '';
  }

  getOptions(defaultText?: string): SelectOption[] {
    return Utils.buildSelectOptions(this.names, defaultText);
  }

  getCodes(): string[] {
    return Object.keys(this.names);
  }

  getNames(): string[] {
    return Object.values(this.names);
  }

  getMapping(): Record<string, string> {
    return { ...this.names };
  }
}
```

## 📝 상수 클래스 작성 방법

### 기본 구조

```typescript
// src/modules/client/constants/client-state.constants.ts
import { BaseCodeConstants } from '@/modules/common/constants/base-code.constants';

export class TaxiClientState extends BaseCodeConstants {
  // 상수 정의
  static readonly OPEN = '1' as const;
  static readonly CLOSE = '2' as const;
  static readonly SHUTDOWN = '3' as const;

  // 이름 매핑
  readonly names = {
    [TaxiClientState.OPEN]: '개업',
    [TaxiClientState.CLOSE]: '폐업',
    [TaxiClientState.SHUTDOWN]: '휴업',
  } as const;

  // 싱글톤 인스턴스 (필요한 경우)
  private static _instance: TaxiClientState;
  static get instance(): TaxiClientState {
    if (!this._instance) {
      this._instance = new TaxiClientState();
    }
    return this._instance;
  }
}
```

## 🔄 사용 방법

### Vue 컴포넌트에서 사용

```vue
<template>
  <div>
    <!-- 선택 옵션으로 사용 -->
    <USelect v-model="selectedState" :options="TaxiClientState.instance.getOptions('상태 선택')" />

    <!-- 상태명 표시 -->
    <span>{{ TaxiClientState.instance.getName(selectedState) }}</span>
  </div>
</template>

<script setup lang="ts">
import { ref, computed } from 'vue';
import { TaxiClientState } from '@/modules/client/constants/client-state.constants';

const selectedState = ref('');

// 선택된 상태의 이름을 계산된 속성으로 사용
const stateName = computed(() => TaxiClientState.instance.getName(selectedState.value));
</script>
```

### Store에서 사용

```typescript
// store에서 상수 사용
import { TaxiClientState } from '@/modules/client/constants/client-state.constants';

export const useClientStore = defineStore('client', () => {
  const clientState = ref('');

  const isOpen = computed(() => clientState.value === TaxiClientState.OPEN);

  const stateOptions = computed(() => TaxiClientState.instance.getOptions('상태 선택'));

  return {
    clientState,
    isOpen,
    stateOptions,
  };
});
```

## 📤 통합 Export

각 모듈의 constants 폴더에는 `index.ts` 파일을 생성하여 모든 상수를 통합 export합니다.

```typescript
// src/modules/client/constants/index.ts
export { TaxiClientState } from './client-state.constants';
export { TaxiClientType } from './client-type.constants';
export { TaxiClientTax } from './client-tax.constants';

// 편의를 위한 인스턴스 export
export const ClientConstants = {
  State: TaxiClientState.instance,
  Type: TaxiClientType.instance,
  Tax: TaxiClientTax.instance,
} as const;
```

## 🚀 마이그레이션 가이드

### 기존 코드에서 새로운 구조로 이전

1. **베이스 클래스 생성**

   ```bash
   # 베이스 클래스 파일 생성
   touch src/modules/common/constants/base-code.constants.ts
   ```

2. **기능별 상수 파일 분리**

   ```bash
   # 각 모듈에 constants 폴더 생성
   mkdir -p src/modules/client/constants
   mkdir -p src/modules/singo/constants
   ```

3. **점진적 마이그레이션**

   - 한 번에 모든 상수를 이전하지 말고 모듈별로 순차적으로 이전
   - 기존 import 경로를 유지하면서 새로운 구조로 점진적 교체
   - 각 단계마다 테스트를 통해 검증

4. **중복 제거**
   - 이전 완료 후 기존 중복 파일들을 제거
   - import 경로를 새로운 구조로 업데이트
